## Steps to Create a Basic Amazon Q App

Creating an Amazon Q application aids in streamlining customer service tasks. Follow these steps to develop a complaint response generator app:

1.  **Initial Setup**
    
    -   Log into the ADS console with necessary credentials.
        
    -   Navigate to the Amazon Q business service.
        
    -   Access the applications section to find your pre-created application.
        
2.  **Locate Deployed URL**
    
    -   Select your application from the list.
        
    -   Locate and use the deployed URL for web experience access.
        
3.  **Authentication**
    
    -   Login through IAM or IAM Identity Center with valid credentials.
4.  **App Creation**
    
    -   In the web experience, click the Apps button.
        
    -   Provide a task description like "generate a polite response to a customer complaint."
        
    -   Hit generate to create the response app.
        
5.  **App Structure**
    
    -   The app includes text input cards for product and complaint details, alongside an output card.
6.  **Testing the App**
    
    -   Test with a sample complaint (e.g., underinflated tennis balls) and assess the response generated.

### Customizing Q Business App for Departmental Requests

Learn to enhance your Q Business app to handle requests from different departments, ensuring appropriate responses for each. Here's a simplified guide:

-   **AWS Console Login**
    
    -   Start by logging into the AWS console with the necessary privileges.
        
    -   Navigate to Amazon Q Business Service and select your application.
        
-   **Web Experience Access**
    
    -   Visit the Deployed URL for the application, logging in with correct IAM credentials if needed.
-   **Application Modification**
    
    -   Access the Customer Complaint Response Generator app.
        
    -   Add a new text input card for department-specific requests.
        
    -   Configure the inputs with a title, placeholder text, and default value, saving changes.
        
-   **Updating Output**
    
    -   Modify the output card to incorporate department-specific responses.
        
    -   Use prompts to ensure responses are tailored from a department member's perspective.
        
-   **Testing**
    
    -   Conduct tests to validate the application generates suitable responses for customer service, marketing, and sales, illustrating varying department strategies.
